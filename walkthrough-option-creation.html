Give me some options people

There are a number of ways of creating an instance of Option<T>, or precisely, an instance of Some<T> or None<T> as Option<T> is abstract.

apply()

Scattered throughout scalesque you will find methods called apply. You can think of these as factory methods or proxies for constructors. In Scala, apply methods allow syntactic sugar, but in c# they are just methods like any other. I have stuck with the Scala naming convention

There is a static class called Option with method called apply with this signature: Option<T> apply<T>(T value). It returns an instance of Some<T> or None<T> based on whether T is null.

Alternately there are two other static methods on Option, Some<T>(T value), and None(). It is left as an exercise for the reader to check what these methods do.

Dealing with libraries that cost you $1bn

As of time of writing, not all libraries in the .net ecosystem use scalesque. This means that you need to defend yourself against null injection attacks.

If you know that a method call can return null, if you call .Opt() the object reference it returns, scalesque will return you an Option<T> and will handle the conversion to Some<T> or None<T> for you.

Scalesque also provides extension methods for known bad guys in the .net base library, like Dictionary.
