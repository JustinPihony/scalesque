Scala inspired functional programming for c#

Introduction

To demonstrate scalesque we will be looking how we could use it to write parts of a web server. Functional programming favours immutable objects, which makes it much easier to write to highly threaded applications like web servers. If data can't be changed you don't need to write as much synchronisation code when accessing it.

Functional programming also favours functions that have no side effects. A side effect would be something like changing the value of a field of a class in a method. It becomes harder understand your code if it performs subtle side effects. Functional programming also favours pure methods. A method is pure if it always returns the same result given the same inputs. Addition is a pure method, DateTime.GetDate() isn't. The more pure methods you can use, the easier your code will be to follow.

Scala and scalesque also make heavy use of the type system and compiler to check correctness of programs. The compiler can't guarantee your program works as you want of course, but we will see later that the type system and compiler can be used instead of standard control flow language statements like 'if' to improve the correctness of your programs. If you can reduce the numbers branching statements in your programs, it becomes easier to understand them, and to see that they are correct.
